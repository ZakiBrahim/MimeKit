ParserOptions Class

Parser options as used by MimeKit.MimeParser as well as various Parse and TryParse methods in MimeKit.

Syntax

public class ParserOptions

Remarks

MimeKit.ParserOptions allows you to change and/or override default parsing options used by methods such as MimeMessage.Load(ParserOptions, System.IO.Stream, System.Threading.CancellationToken) and others.

Remarks

By default, new instances of MimeKit.ParserOptions enable rfc2047 work-arounds (which are needed for maximum interoperability with mail software used in the wild) and do not respect the Content-Length header value.

AddressParserComplianceMode Property

Gets or sets the compliance mode that should be used when parsing rfc822 addresses.

Syntax

public RfcComplianceMode AddressParserComplianceMode { get; set; }

Value

The RFC compliance mode.

Remarks

In general, you'll probably want this value to be RfcComplianceMode.Loose (the default) as it allows maximum interoperability with existing (broken) mail clients and other mail software such as sloppily written perl scripts (aka spambots).

It should be noted that even in RfcComplianceMode.Strict mode, the address parser is fairly liberal in what it accepts. Setting it to RfcComplianceMode.Loose just makes it try harder to deal with garbage input.

CharsetEncoding Property

Gets or sets the charset encoding to use as a fallback for 8bit headers.

Syntax

public System.Text.Encoding CharsetEncoding { get; set; }

Value

The charset encoding.

Remarks

MimeKit.Utils.Rfc2047.DecodeText(ParserOptions, Byte[]) and MimeKit.Utils.Rfc2047.DecodePhrase(ParserOptions, Byte[]) use this charset encoding as a fallback when decoding 8bit text into unicode. The first charset encoding attempted is UTF-8, followed by this charset encoding, before finally falling back to iso-8859-1.

Exceptions

Type Reason

-or-

ArgumentException

type is not a subclass of MimeKit.Multipart, MimeKit.MimePart, or MimeKit.MessagePart.

-or-

type does not have a constructor that takes only a MimeKit.MimeEntityConstructorInfo argument.

Remarks

Your custom MimeKit.MimeEntity class should not subclass MimeKit.MimeEntity directly, but rather it should subclass MimeKit.Multipart, MimeKit.MimePart, MimeKit.MessagePart, or one of their derivatives.
